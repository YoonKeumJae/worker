# 릴리스 Runbook

## 목적

`worker` 앱을 배포 가능한 상태로 만들 때 따를 절차를 기록한다.

현재는 실제 배포 방식이 확정되지 않았다. 이 문서는 초기 운영 기준이며, 배포 방식 확정 후 갱신한다.

## 릴리스 전 확인

- `main` 브랜치가 최신인지 확인
- 모든 PR이 병합되었는지 확인
- CI `Validate`가 통과했는지 확인
- 기능 문서와 README가 최신인지 확인
- 앱 버전이 갱신되었는지 확인
- macOS 빌드가 성공했는지 확인

## 예상 명령

```bash
pnpm install
pnpm test
pnpm build
pnpm tauri build
```

실제 명령은 scaffold 후 갱신한다.

## 릴리스 기록

릴리스 시 다음을 기록한다.

- 버전
- 날짜
- 포함 기능
- 수정 사항
- 알려진 제한
- 다운로드/배포 위치

## 롤백

초기 버전에서는 자동 롤백 절차가 없다.

문제가 있으면:

1. 문제가 있는 릴리스를 배포 중단한다.
2. 직전 정상 빌드를 다시 배포한다.
3. 원인을 issue 또는 문서에 기록한다.
4. 수정 PR을 만든다.

## 미정 사항

- 버전 규칙
- 코드 서명
- notarization
- 배포 채널
- 자동 업데이트 지원 여부

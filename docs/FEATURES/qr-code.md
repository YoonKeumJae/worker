# QR코드 생성 기능 명세

## 목적

URL을 QR코드로 변환해 업무 중 빠르게 공유할 수 있게 한다.

## 사용자

- 개인 사용자
- URL을 QR코드 이미지로 저장하거나 다른 앱에 붙여넣고 싶은 사용자

## 범위

현재 구현:

- URL 입력
- URL 형식 검증
- QR코드 미리보기

후속 구현:

- PNG 저장
- SVG 저장
- 이미지 클립보드 복사
- 성공/실패 상태 표시

첫 버전 제외:

- 색상 커스터마이징
- 로고 삽입
- QR error correction level 설정 UI
- Wi-Fi QR
- 연락처 QR
- 대량 생성
- QR 스캔 기능

## 입력

- URL 문자열

검증 기준:

- 빈 값은 허용하지 않는다.
- `http://` 또는 `https://` URL을 우선 지원한다.
- 잘못된 URL은 QR 생성 action을 실행하지 않는다.

오류 메시지:

- `URL을 입력하세요.`
- `유효한 URL을 입력하세요.`

## 출력

- 화면 미리보기 QR
- PNG 파일
- SVG 파일
- 클립보드 이미지

## UX 흐름

1. 사용자가 URL을 입력한다.
2. 유효한 URL이면 QR 미리보기를 생성한다.
3. 현재 구현은 화면에서 QR 미리보기를 표시한다.
4. 후속 구현에서 사용자는 `PNG 저장`, `SVG 저장`, `이미지 복사` 중 하나를 선택한다.
5. 작업 성공 시 짧은 성공 메시지를 표시한다.
6. 실패 시 원인과 다음 행동을 표시한다.

## 저장

PNG 저장:

- 사용자가 저장 위치와 파일명을 선택한다.
- 기본 파일명은 URL host를 기반으로 만들 수 있다.
- 저장 실패 시 원인을 표시한다.

SVG 저장:

- 사용자가 저장 위치와 파일명을 선택한다.
- QR 벡터 데이터가 손상되지 않아야 한다.

## 클립보드

- `이미지 복사`는 QR 이미지를 클립보드에 저장한다.
- 복사 성공 후 `복사 완료` 상태를 표시한다.
- 지원되지 않는 환경 또는 실패 시 오류를 표시한다.

## 접근성

- URL 입력 field는 명확한 label을 가진다.
- 저장/복사 버튼은 키보드로 접근 가능해야 한다.
- 오류 메시지는 입력 field와 연결되어야 한다.
- QR 미리보기는 고대비 배경에서 표시한다.

## 테스트 기준

- 빈 URL 입력 시 오류 표시
- 잘못된 URL 입력 시 오류 표시
- 정상 URL 입력 시 QR 미리보기 생성

후속 테스트 기준:

- PNG 저장 성공
- SVG 저장 성공
- 이미지 복사 성공
- 저장 취소 시 오류가 아니라 취소 상태로 처리

## 확정 사항

- QR 미리보기 생성 라이브러리는 `qrcode.react`를 사용한다.

## 미정 사항

- 기본 파일명 규칙
- 클립보드 처리를 UI/Rust 중 어디서 담당할지
